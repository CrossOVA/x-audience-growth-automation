# Автоматизация роста аудитории в X

Сценарий n8n для поиска релевантных пользователей в X, аккуратного взаимодействия с ними через API, передачи найденного паразитного контента в дальнейший контент-завод.

## Что делает сценарий

Сценарий регулярно ищет свежие публикации по заданным запросам, достаёт авторов и контент, проверяет историю взаимодействий и выполняет действия только с теми пользователями, которые ещё не были обработаны.

Задача проекта – убрать ручную рутину из поиска аудитории, но сохранить контроль: не повторяться, не спамить одних и тех же людей, действовать непредсказуемо, учитывать историю и логировать действия.

## Что реализовано

- регулярный запуск по расписанию;
- поиск свежих публикаций через X API;
- получение данных об авторах;
- проверка пользователей в Supabase;
- исключение уже обработанных профилей;
- случайная задержка перед действиями;
- подписки / лайки через X API;
- передача информации из публикаций в дальнейший поток;
- запись результата в базу;
- обработка ошибок без падения всего сценария.

## Мой вклад

Разработал архитектуру и собрал сценария автоматизации, настроил работу с X API, написал JavaScript-логику для подготовки ID пользователей, проверки истории действий и ограничения повторных взаимодействий.

Также я настроил связь с Supabase, добавил задержки и обработку ошибок, чтобы сценарий работал предсказуемо, а не как хаотичный бот.
